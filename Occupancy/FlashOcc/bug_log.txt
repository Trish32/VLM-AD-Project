ARCHITECTURE (single keyframe, 6 cams, 256x704)
-----------------------------------------------
ResNet-50 (out_indices 2,3 -> 1024@s16, 2048@s32)
  -> CustomFPN (in [1024,2048], out 256, out_ids[0], NO norm/act) -> 256@16x44
  -> LSSViewTransformer (depth_net Conv2d 256->88+64; D=88 depth bins
       [1,45,0.5]; collapse_z) -> BEV (B,64,200,200)
  -> CustomResNet BEV backbone (BasicBlock, numC 64->[128,256,512], stride 2,2,2)
  -> FPN_LSS (in 512+128=640, out 256, up x4 + extra up x2) -> (B,256,200,200)
  -> BEVOCCHead2D (final_conv 256->256 3x3; predicter Lin256->512 +Softplus
       +Lin512->288) -> (B,200,200,16,18) class logits

KEY NON-OBVIOUS FACTS
---------------------
1. bev_pool_v2 (the only CUDA op) is just a depth-weighted scatter-add:
     bev[ranks_bev] += depth[ranks_depth] * context_feat[ranks_feat]
   Reimplemented with Tensor.index_add_ (MPS-OK). No cumsum trick / interval
   bookkeeping needed (those only matter to the custom kernel). Verified against
   the official test_bev_pool_v2 case (sum == 4.4 with C=2).
   File: model/view_transformer.py::LSSViewTransformer.voxel_pooling

2. img_backbone uses torchvision ResNet-50's EXACT key layout (conv1, bn1,
   layer{1..4}.{i}.conv{1,2,3}, downsample.{0,1}) — so torchvision resnet50()
   loads the checkpoint directly with no remapping.

3. CustomResNet (BEV encoder) downsample branch is a SINGLE 3x3 conv WITH bias
   (not 1x1 conv+BN) -> checkpoint keys are layers.{s}.0.downsample.{weight,bias}.
   BasicBlock naming conv1/bn1/conv2/bn2 matches mmdet.

4. CustomFPN / occ_head ConvModules have NO norm and NO activation here
   (norm_cfg=act_cfg=None in config) -> wrap a bare Conv2d as `.conv` so keys
   `<name>.conv.weight/bias` map. FPN top-down add uses NEAREST interpolation.

5. Test-time image preprocessing (PrepareImageInputs test branch):
   resize = 704/1600 = 0.44 -> (704,396); crop (0,140,704,396) -> 704x256.
   post_rot = diag(0.44,0.44,1); post_tran = (0,-140,0); bda = I(3).

6. Normalisation reproduces mmcv imnormalize(to_rgb=True) on an already-RGB PIL
   image: it REVERSES the channel order (cvtColor BGR<->RGB) BEFORE subtracting
   the ImageNet mean. Must replicate the swap (img[...,::-1]) — the checkpoint
   was trained with it. File: data/loader.py::mmlab_normalize

7. Geometry (prepare_inputs sensor2keyego, get_ego_coor frustum->BEV) uses 4x4
   matrix inverses in float64. MPS has NO float64, so all geometry runs on CPU
   (.cpu().double()) and only the conv stack + index_add scatter run on-device.
   prepare_inputs maps every cam into camera-0's ego frame:
     sensor2keyego = inv(ego2global[:,0]) @ ego2global @ sensor2ego

8. Output axis convention: head permute(0,3,2,1) gives (B,Dx,Dy,C); after Dz
   expansion -> (B,Dx,Dy,Dz=16,18). Matches GT (Dx,Dy,Dz). get_occ = softmax
   over classes then argmax. Class 17 = 'free'.

RAYIOU (added) — surface metric, pure-PyTorch raycaster
-------------------------------------------------------
eval.py --metric ray-iou adds the SparseOcc/Occ3D RayIoU alongside mIoU.
Reimplements ray_metrics.py WITHOUT the CUDA dvr renderer:
- generate_lidar_rays(): ~15840 fixed (pitch x azimuth) directions.
- LiDAR ego origins from the devkit (loader.get_lidar_origins) replicate
  EgoPoseDataset: up to 8 per-scene viewpoints in the ref ego frame, |x|,|y|<39.
- raycast() (tools/ray_metrics.py): marches each ray in fixed metric steps
  (default 0.2 m = half voxel), takes the FIRST occupied sample -> (class,
  distance). Dense-sampling approximation of voxel-DDA; on contiguous surfaces
  matches DDA well within the 1 m threshold. Runs on MPS (floor/gather/argmax).
- RayIoU_c = TP/(GT+pred-TP) over rays whose GT hit is non-free; TP needs
  class match AND |depth_pred-depth_gt| < thr; mean over thr {1,2,4} & 17 cls.
- scene-0103: RayIoU 14.1 (@1 9.5/@2 14.1/@4 18.6) vs mIoU 23.27. RayIoU is
  stricter (needs correct surface DEPTH, not just voxel overlap). Telling:
  driveable_surface mIoU ~87 but RayIoU@1 22.5 -> @4 64 (grazing-angle ground
  rays are depth-sensitive; mIoU hides this).
- ~1.4 it/s on MPS (2 volumes x 8 origins x 15840 rays per sample).
